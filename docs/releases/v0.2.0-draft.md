# L2MAS v0.2.0 Draft

This draft tracks the provider adapter upgrade planned after v0.1.0. It is not a release announcement until the repository is tagged and the verification matrix is updated with fresh results.

## Highlights

- Provider registry entries now carry status metadata: `verified`, `experimental`, `template`, or `mock`.
- Provider router supports contract-tested adapter boundaries for OpenAI-compatible chat, Ollama, ComfyUI, generic REST, ElevenLabs TTS, MCP template, mock providers, and local FFmpeg.
- Animation pipeline routes voice, motion, and composition through provider adapters and records fallback warnings.
- Optional live provider tests are gated by explicit `L2MAS_LIVE_*` environment variables.

## Verification Policy

Only providers listed as `verified` in [docs/provider-verification.md](../provider-verification.md) may be described as live-validated. Experimental and template providers are adapter contracts until a real service is tested and recorded.
