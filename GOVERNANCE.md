# Governance

L2MAS currently uses a lightweight maintainer-led governance model.

## Maintainers

Maintainers are responsible for:

- reviewing pull requests
- triaging issues
- protecting the main branch
- enforcing the Code of Conduct
- keeping licensing and third-party notices accurate

## Decision Making

Project decisions should favor:

- open protocols over vendor lock-in
- provider registry based integration over hard-coded model choices
- reproducible MVP paths before distributed v2.0 complexity
- clear licensing boundaries for third-party runtimes, models, datasets, and media assets

For major architecture changes, open an issue or discussion before implementation so contributors can compare tradeoffs.
